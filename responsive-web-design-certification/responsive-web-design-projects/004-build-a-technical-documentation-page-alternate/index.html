Making a valid <img> tag

<img src="https://upload.wikimedia.org/wikipedia/commons/b/b6/New_Map_of_Great_Britain_and_Ireland_%281806%29.jpg" alt="Closeup on New Map of Great Britain (1806)">

align

Not valid in HTML 5. Use CSS vertical-align or float instead. In HTML 4.01, this attribute would align an image vertically or horizontally.

  • Use the CSS property vertical-align: top; instead of align="top"
  • Use the CSS property vertical-align: middle; instead of align="middle"
  • Use the CSS property vertical-align: bottom; instead of align="bottom"
  • Use the CSS property float: left; instead of align="left"
  • Use the CSS property float: right; instead of align="right"

alt

Required in HTML 5. Shows alternate text in case the image cannot be displayed.

Alternate text for the alt tag is not the same thing as a tooltip; use the title attribute (which can be used on any HTML element) to display a tooltip.

border

Not valid in HTML 5. Use CSS border instead. In HTML 4.01, this attribute will draw a border around the image of x pixels.

Since I set the DOCTYPE of this pen to HTML 5, CodePen will throw an HTML Inspector warning: "The 'border' attribute is no longer valid on the <img> element and should not be used." and the browser will not display any border.

crossorigin

I read quite a lot to try to wrap my head around this one. I think that crossorigin is intended to act as a security tool. In a JavaScript app, unless all resources (including images) from different domains have properly set their crossorigins, the browser should treat its canvas as "tainted" for security purposes. The document itself can display crossorigin resources to the browser canvas and the JavaScript can write to the canvas, but if the JavaScript tries to read data back from a "tainted" canvas, the browser will throw a security error. In effect, this prevents things like a malicious page element from taking a screenshot of every page you browse.

I want to explore the whole topic of CORS, since it seems it would be very relevant to any nontrivial web app. Insofar as CORS concerns the <img> tag, however, I think it is more of a "when I obviously need it" kind of learning.

crossorigin can take a value of anonymous or use-credentials.

hspace

Not valid in HTML 5. Can work in modern browsers, but use CSS margin instead. In HTML 4.01, this attribute will add a whitespace of x pixels to the left and right side of the image.

Since I set the DOCTYPE of this pen to HTML 5, CodePen will throw an HTML Inspector warning: "The 'hspace' attribute is no longer valid on the <img> element and should not be used," but the browser displays appropriate whitespace anyway (Chrome 71).

ismap

This tag is a boolean value: if it is present, the image is a server-side image map. When the image is wrapped in an appropriate <a> tag, the browser sends the clicked coordinates to the server linked in the <a> tag as a URL query string. For example, the following code:

<a href="mysite.example/example.php">
    <img src="mysite.example/image.jpg" alt="This is not an image" ismap>
  </a>

If you clicked on coordinates, say, 100 by 100, the browser will treat it like a link to http://mysite.example/example.php?100,100, and the server will process that URL query accordingly.

I'm not sure this is actually used much at all on the modern web.

longdesc

This tag takes a URL as its value. It was intended as an accessiblity enhancement that can link to an extended description of an image, particularly useful for screen readers. However, it is not currently supported by any major browser vendors, and most sources recommended aria-describedby as the replacement for longdesc.

sizes

sizes, together with srcset, is immensely useful for creating responsive websites. You can create custom rules to specify a different image for mobile, desktop, device orientation, HiDPI, etc. sizes takes media conditions as arguments (similar to @media queries). Media conditions can even be combined using and, not, and or. However, you must specify each image's width value in srcset to use sizes and its custom media conditions.

This simple example code would make an image take up 100% of the browser viewport if held in portrait mode, or 80% if held in landscape mode. The browser will automatically choose an appropriately-sized image based on its calculations of the actual viewport size, or if it is a browser which does not support sizes and srcset, it will fall back to using the default specified by src:

<img src="mysite.example/image-default.jpg"
    alt="This is not an image"
    srcset="image-small.jpg 240w, image-medium.jpg 960w, image-large.jpg 1920w"
    sizes="(orientation: landscape) 80vw,
           100vw;>

That example is pretty simple &emdash; so simple it would be better to not use those custom attributes at all, and instead use CSS. When an image would need to be materially different, not just scaled up and down, sizes becomes very useful.

For example, an author's portrait in an article can focus on her face on a small mobile screen, stretch to include the background when holding a mobile in landcape, then pull out to a full body shot in tablet or desktop:

<img src="author-default.jpg" alt="Edna Hollis, author of All Our Dreams"
    srcset="author-mobile.jpg 500w, author-mobile-wide.jpg 1000w, author-huge.jpg 3840w"
    sizes="(max-width: 480px) and (orientation: portrait) 80vw;
           (orientation: landscape) 100vw"

src

Required in HTML 5. Specifies an image's URL.

srcset

srcset takes two arguments for each image: the image URL and either its pixel density (1x, 2x, etc.) or width in pixels (e.g., 120w for an image 120 pixels wide). The browser will choose an image automatically appropriate for the device or current viewport. srcset is supported by just shy of 90% of browsers, and unsupported browsers will display the default image specified by src.

You could use srcset on its own to get a certain amount of responsiveness for free. You can pair it with sizes to direct the browser to display specific image files in response to detailed media conditions (similar syntax to media queries).

usemap

This tag points to the appropriate map tag in the document to give the browser information to build a client-side image map. Roughly, it is custom linked areas inside a particular image, rather than just one link per image or (in the bad old days) cutting what should be a single image into many pieces. Here is an example of a client-side image map:

<img src="mysite.example/image.jpg" alt="This is not an image either" usemap="#1996">
  <map name="1996">
    <area shape="circle" coords="15,15,2" href="heads.html" alt="Heads I Win">
    <area shape="rect" coords="40,40,100,80" href="" alt="Tails You Lose">
    <area shape="poly" coords="50,40,20,20,11,140,444,290" href="" alt="King Me">
  </map>

Notice that there is alt text, unlike for ismap, and that there are multiple ways to build shapes, including complex polygons (though I just entered random numbers for this example).

Like ismap, I'm not sure this is used much on the modern web either.

vspace

Not valid in HTML 5. Can work in modern browsers, but use CSS margin instead. In HTML 4.01, this attribute will add a whitespace of x pixels to the top and bottom of the image.
